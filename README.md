# HEI Network Timeline

A static [Next.js](https://nextjs.org) site that lists HEI Network releases in a vertical timeline ordered by **air date**. Data lives in [`data/timeline.json`](data/timeline.json) (generated by the scraper). A single endpoint serves the same JSON: **`GET /api/timeline`**.

- **Official site**: [heinetwork.tv](https://www.heinetwork.tv)
- **Data refresh**: run the Python scraper locally, commit `data/timeline.json`, push (see [`scraper/README.md`](scraper/README.md)).

## Development

```bash
nvm use   # optional — see .nvmrc
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). API: [http://localhost:3000/api/timeline](http://localhost:3000/api/timeline).

## Deploy to Vercel

1. Push this repo to GitHub/GitLab/Bitbucket.
2. In [Vercel](https://vercel.com), **Add New Project** → import the repo (framework: Next.js is auto-detected).
3. **Environment variable** (optional but recommended for correct Open Graph URLs):
   - `NEXT_PUBLIC_SITE_URL` = `https://your-domain.com` (no trailing slash)
4. Deploy. Add your custom domain under **Project → Settings → Domains** (follow the DNS / CNAME instructions Vercel shows).

No database or server secrets required.

## Seeded data vs live API

If you don’t have network access when running the scraper (e.g. CI), the repo may ship with a **`data/timeline.json`** seeded from your existing archive. To replace it with live WordPress data, run:

```bash
cd scraper && pip install -r requirements.txt && python scrape.py
```

from a machine that can reach `https://www.heinetwork.tv/wp-json/`.
